# Marth's Enchanting Overhaul (MEO)

A full replacement for Skyrim SE enchanting that turns enchantments into
**gems**: socketed into your own equipment from a pouch menu anywhere, and
**leveled through use**. Weapons, armor, and jewelry gain sockets; staves,
bound weapons, and scripted artifacts are detected and left alone.

Built entirely from Linux with no Creation Kit and no xEdit: the ESP is
generated byte-by-byte from Python, the gem catalog is parsed straight out of
the game masters, and the native SKSE plugin (CommonLibSSE-NG) builds on
GitHub Actions.

> **Status: in active development, playable.** The native core ships and is
> validated in-game through `v0.25.x`: 51 gem families (weapon + armor),
> multi-socket, gem leveling/birthing, enchanting-station soul feeding, a
> perk layer, and an MCM. See **Roadmap** below and `Docs/INDEX.md`.

## The idea

- **Enchantments become gems.** Every generic enchantment collapses into a
  gem *family* (the catalog is generated from ~740 ENCH records in the game
  masters, deduplicated by effect). Weapon gems are on-hit effects; armor
  gems are constant effects. 51 families ship, each at levels I–V.
- **Socket anywhere, lose nothing.** The Gem Pouch power opens the socket
  menu with no crafting station. Socketing and unsocketing your own gems is
  always free, lossless, and reversible — an unsocketed gem returns as *the
  same gem*, carrying its own level and banked XP.
- **Gems level I → V through your kills**, then a mastered gem *births* a
  fresh level-I copy of itself — the only way to replicate one. Enchanting
  stations get a second life: feed filled soul gems to a socketed gem for XP.
- **Sockets scale with skill.** Gloves are dual-socket by default; perks add
  a second linked socket to chest armor and weapons. A linked pair means both
  gems ride one item — and both earn XP from every kill.
- **Gems never drain.** No charge, no recharging. The trade is that power
  comes from the gem's level, which you earn.
- **The item is the database.** Socket state lives on each item instance and
  in the SKSE co-save — no Papyrus save bloat, no side-table to desync.

## Architecture

- **`MEO.esp`** — generated by `MEO_GenerateESP.py` (never hand-edited):
  gem MISC items, magic-effect markers, the pouch power, perk records, MCM
  quest. FormIDs are frozen once shipped; the generator is freeze-aware.
- **`MEO.dll`** — CommonLibSSE-NG SKSE plugin, built on GitHub Actions
  (windows-latest, MSVC): the socket engine, created-enchantment stamping,
  kill-XP and leveling, the in-game ImGui gem menu, perk effects, and
  versioned co-save serialization. Address-Library-based.
- **MCM** — MCM Helper config paired with an INI; every gameplay tunable
  (drop rates, XP rates, power scale) is live-editable in game.
- Papyrus is compile-time only (one empty MCM stub), compiled under Proton
  wine from Linux.

## Repo layout

```
MEO_GenerateESP.py   generates MEO.esp (never hand-edited)
native/              the SKSE plugin (plugin.cpp, CMake, vcpkg, CI-built)
data/                gem catalog parsed from the game masters (JSON)
tools/               catalog parser, header generator, compile.sh, release script
Source/Scripts/      MCM stub source (compile-only)
Docs/                START AT Docs/INDEX.md
releases/vX.Y.Z/     immutable release archive (complete standalone zips)
```

## Documentation

**`Docs/INDEX.md` is the entry point.** The docs are written so any capable
person or model can continue development from them alone: the design spec
(`DESIGN.md`), balance math (`BALANCE.md`), verified engine mechanisms
(`ENGINE_NOTES.md`), a binary-format guide, a debugging cookbook, and
in-game test guides.

## Roadmap

- **Done** — native socket core; 51 gem families with armor slots; gem
  leveling, birthing, and per-instance XP; multi-socket with perk-gated
  second sockets; enchanting-station soul feeding and gem destruction;
  world/loot gem spawns with a tier rarity curve; perk effects; MCM; on-load
  reactivation of worn sockets.
- **Next** — install-time C# (Mutagen) tool: scans the load order, replaces
  its enchanting perk tree with MEO's perks, and handles enchanted-loot
  conversion edge cases.
- **Then** — elemental-affinity and gem-class perks, support/linked gem
  behaviors, FOMOD packaging, wider load-order compatibility passes.

## Requirements

SKSE64, SkyUI, MCM Helper, Address Library. Developed and tested against
Skyrim SE 1.6.1170 under a Requiem-based list (LoreRim); core targets vanilla
SE + SKSE.
